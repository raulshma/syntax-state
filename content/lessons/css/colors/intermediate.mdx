## Gradients

Gradients let you create smooth transitions between two or more colors. They're perfect for backgrounds, buttons, and creating depth in your designs.

### Linear Gradients

Linear gradients transition colors along a straight line. Think of it like a sunset - colors blend smoothly from one to another.

```css
.gradient-box {
  background: linear-gradient(to right, #3B82F6, #8B5CF6);
}

.vertical-gradient {
  background: linear-gradient(to bottom, #EF4444, #F59E0B);
}

.diagonal-gradient {
  background: linear-gradient(45deg, #10B981, #3B82F6);
}
```

<KeyConcept title="Linear Gradient Syntax">
```css
linear-gradient(direction, color1, color2, ...)
```

**Directions:**
- `to right`, `to left`, `to top`, `to bottom`
- Angles: `45deg`, `90deg`, `180deg`
- Default: `to bottom`
</KeyConcept>

### Color Stops

Control exactly where colors transition by adding **color stops** (percentages or lengths).

```css
.custom-gradient {
  background: linear-gradient(
    to right,
    #3B82F6 0%,
    #8B5CF6 50%,
    #EC4899 100%
  );
}

.sharp-transition {
  background: linear-gradient(
    to right,
    #3B82F6 0%,
    #3B82F6 50%,
    #EC4899 50%,
    #EC4899 100%
  );
}
```

<InfoBox type="tip" title="Creating Sharp Transitions">
To create a sharp line instead of a smooth blend, place two color stops at the same position (like 50% in the example above).
</InfoBox>

### Radial Gradients

Radial gradients radiate outward from a center point, like ripples in water or a spotlight effect.

```css
.radial-gradient {
  background: radial-gradient(circle, #3B82F6, #1E40AF);
}

.ellipse-gradient {
  background: radial-gradient(ellipse, #EC4899, #BE185D);
}

.positioned-gradient {
  background: radial-gradient(
    circle at top left,
    #10B981,
    #059669
  );
}
```

<KeyConcept title="Radial Gradient Syntax">
```css
radial-gradient(shape at position, color1, color2, ...)
```

**Shapes:** `circle` or `ellipse` (default)
**Positions:** `at center`, `at top left`, `at 30% 70%`
</KeyConcept>

### Conic Gradients

Conic gradients rotate around a center point, perfect for pie charts or color wheels.

```css
.conic-gradient {
  background: conic-gradient(
    from 0deg,
    #EF4444,
    #F59E0B,
    #10B981,
    #3B82F6,
    #8B5CF6,
    #EF4444
  );
}

.pie-chart {
  background: conic-gradient(
    #3B82F6 0deg 120deg,
    #10B981 120deg 240deg,
    #F59E0B 240deg 360deg
  );
  border-radius: 50%;
}
```

## Color Functions

Modern CSS provides powerful functions for manipulating colors dynamically.

### currentColor

The `currentColor` keyword uses the current text color value. It's perfect for creating consistent designs.

```css
.icon-button {
  color: #3B82F6;
  border: 2px solid currentColor;
  /* border will be #3B82F6 */
}

.icon-button:hover {
  color: #1E40AF;
  /* border automatically updates to #1E40AF */
}
```

<KeyConcept title="Why Use currentColor?">
`currentColor` creates a relationship between text color and other properties. Change the color once, and related properties update automatically!
</KeyConcept>

### CSS Variables for Colors

CSS custom properties (variables) make color management much easier, especially for themes.

```css
:root {
  --primary-color: #3B82F6;
  --primary-dark: #1E40AF;
  --primary-light: #93C5FD;
}

.button {
  background-color: var(--primary-color);
  border: 2px solid var(--primary-dark);
}

.button:hover {
  background-color: var(--primary-dark);
}
```

<InfoBox type="tip" title="Color System Best Practice">
Define your color palette as CSS variables at the `:root` level. This makes it easy to maintain consistency and create theme switchers!
</InfoBox>

## Color Accessibility

Creating accessible color combinations ensures everyone can use your website, including people with visual impairments.

### Contrast Ratios

The **contrast ratio** measures the difference between text and background colors. WCAG (Web Content Accessibility Guidelines) defines minimum ratios:

<KeyConcept title="WCAG Contrast Requirements">
**Level AA (Minimum):**
- Normal text: 4.5:1 contrast ratio
- Large text (18pt+ or 14pt+ bold): 3:1 contrast ratio

**Level AAA (Enhanced):**
- Normal text: 7:1 contrast ratio
- Large text: 4.5:1 contrast ratio
</KeyConcept>

### Good vs Bad Contrast Examples

```css
/* ‚ùå Poor contrast - hard to read */
.bad-contrast {
  color: #CCCCCC;
  background-color: #FFFFFF;
  /* Ratio: 1.6:1 - fails WCAG */
}

/* ‚úÖ Good contrast - easy to read */
.good-contrast {
  color: #1F2937;
  background-color: #FFFFFF;
  /* Ratio: 16.1:1 - passes AAA */
}

/* ‚úÖ Also good */
.good-contrast-2 {
  color: #FFFFFF;
  background-color: #1E40AF;
  /* Ratio: 8.6:1 - passes AAA */
}
```

<InfoBox type="warning" title="Common Accessibility Mistakes">
- Light gray text on white backgrounds
- Yellow text on white backgrounds
- Relying only on color to convey information
- Using low-contrast colors for important UI elements
</InfoBox>

### Color Blindness Considerations

About 8% of men and 0.5% of women have some form of color blindness. Design tips:

1. **Don't rely on color alone**: Use icons, patterns, or text labels
2. **Test your palette**: Use tools to simulate color blindness
3. **Avoid problematic combinations**:
   - Red and green (most common color blindness)
   - Blue and yellow (less common but still important)

## Summary

You've learned powerful techniques for working with colors in CSS!

<KeyConcept title="Key Takeaways">
**Gradients:**
- Linear gradients: Smooth transitions along a line
- Radial gradients: Transitions radiating from a center
- Conic gradients: Rotational color transitions
- Use color stops for precise control

**Color Functions:**
- `currentColor`: Inherits the current text color
- CSS variables: Create maintainable color systems
- Define color palettes at `:root` for consistency

**Accessibility:**
- Maintain 4.5:1 contrast ratio for normal text (AA)
- Maintain 7:1 contrast ratio for enhanced accessibility (AAA)
- Don't rely on color alone to convey information
- Consider color blindness when choosing palettes
- Use tools to test your color combinations
</KeyConcept>
