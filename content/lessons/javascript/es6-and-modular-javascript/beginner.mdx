## let and const: Better Variables

Remember how we used `var` to create variables? ES6 gives us two better options: `let` and `const`.

Think of it like organizing your room:
- **const** = A permanent shelf that never moves (for things that won't change)
- **let** = A movable box (for things that might change)
- **var** = The old messy way (avoid this!)

### const: For Things That Don't Change

```javascript
const PI = 3.14159;        // Mathematical constant
const APP_NAME = "MyApp";  // App configuration
const birthday = "1990-01-15";  // Your birthday won't change!

// This will cause an error:
// PI = 3.14;  // ❌ Can't reassign a const!
```

### let: For Things That Change

```javascript
let score = 0;
score = 10;  // ✅ This works!
score = score + 5;  // ✅ Now score is 15

let userName = "Guest";
userName = "Alice";  // ✅ Can change
```

<KeyConcept title="When to Use Which?">
- Use **const** by default — it's safer!
- Use **let** only when you need to change the value
- Never use **var** in modern code
</KeyConcept>

## Arrow Functions: Shorter Function Syntax

Arrow functions are a shorter way to write functions. They use the `=>` symbol (which looks like an arrow!).

### The Old Way vs The New Way

```javascript
// Old way (ES5)
function add(a, b) {
  return a + b;
}

// New way (ES6) - Arrow function
const add = (a, b) => a + b;
```

Both do the same thing, but the arrow function is much shorter!

### Different Arrow Function Styles

```javascript
// One parameter - no parentheses needed
const double = n => n * 2;

// Multiple parameters - need parentheses
const add = (a, b) => a + b;

// No parameters - empty parentheses
const sayHello = () => "Hello!";

// Multiple lines - need curly braces and return
const greet = (name) => {
  const message = `Hello, ${name}!`;
  return message;
};
```

<InfoBox type="info" title="When to Use Arrow Functions">
Arrow functions are great for:
- Short, simple functions
- Callbacks (functions passed to other functions)
- Array methods like map, filter, reduce
</InfoBox>

<CodePlayground
  initialCode={`// Arrow functions in action

// Double each number

console.log("Doubled:", doubled);

// Filter even numbers
const evens = numbers.filter(n => n % 2 === 0);
console.log("Evens:", evens);

// Sum all numbers
const sum = numbers.reduce((total, n) => total + n, 0);
console.log("Sum:", sum);`}

## Template Literals: Easy String Building

Template literals use backticks (\`) instead of quotes and let you embed variables directly in strings!

### The Old Way vs The New Way

```javascript
// Old way - messy string concatenation
const name = "Alice";
const age = 25;
const message = "Hello, " + name + "! You are " + age + " years old.";

// New way - clean template literal
const message = `Hello, ${name}! You are ${age} years old.`;
```

### Multi-line Strings

```javascript
// Old way - awkward
const html = "<div>\n" +
  "  <h1>Title</h1>\n" +
  "</div>";

// New way - natural
const html = `
  <div>
    <h1>Title</h1>
  </div>
`;
```

<KeyConcept title="Template Literal Rules">
- Use backticks (\`) not regular quotes
- Put variables inside `${...}`
- You can put any expression inside `${...}`, not just variables!
</KeyConcept>

## Destructuring: Unpacking Made Easy

Destructuring lets you "unpack" values from arrays and objects into separate variables. It's like opening a gift box and taking out each item!

### Object Destructuring

```javascript
// Without destructuring
const person = { name: "Alice", age: 25, city: "NYC" };
const name = person.name;
const age = person.age;

// With destructuring - much cleaner!
const { name, age, city } = person;
console.log(name);  // "Alice"
console.log(age);   // 25
```

### Array Destructuring

```javascript
// Without destructuring
const colors = ["red", "green", "blue"];
const first = colors[0];
const second = colors[1];

// With destructuring
const [first, second, third] = colors;
console.log(first);   // "red"
console.log(second);  // "green"
```

## Spread and Rest: The Three Dots

The `...` operator has two uses:
- **Spread**: Expands an array or object
- **Rest**: Collects multiple items into one

### Spread Operator

```javascript
// Copying arrays
const original = [1, 2, 3];
const copy = [...original];

// Combining arrays
const arr1 = [1, 2];
const arr2 = [3, 4];
const combined = [...arr1, ...arr2];  // [1, 2, 3, 4]

// Copying objects
const user = { name: "Alice", age: 25 };
const userCopy = { ...user };

// Extending objects
const extended = { ...user, city: "NYC" };
```

### Rest Parameters

```javascript
// Collect remaining arguments
function sum(...numbers) {
  return numbers.reduce((a, b) => a + b, 0);
}

sum(1, 2);        // 3
sum(1, 2, 3, 4);  // 10
```

## ES Modules: Organizing Your Code

As your code grows, you'll want to split it into separate files. ES modules let you share code between files using `import` and `export`.

### Exporting Code

```javascript
// math.js - Export functions and values
export const PI = 3.14159;

export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

// Default export (one per file)
export default class Calculator {
  // ...
}
```

### Importing Code

```javascript
// main.js - Import what you need
import Calculator, { PI, add } from './math.js';

console.log(PI);        // 3.14159
console.log(add(2, 3)); // 5
```

## Key Takeaways

<KeyConcept title="const and let">
Use `const` for values that don't change, `let` for values that do. Forget about `var`!
</KeyConcept>

<KeyConcept title="Arrow Functions">
Shorter syntax for functions: `const add = (a, b) => a + b;`
</KeyConcept>

<KeyConcept title="Template Literals">
Use backticks and `${...}` for easy string building: `` `Hello, ${name}!` ``
</KeyConcept>

<KeyConcept title="Destructuring">
Unpack values easily: `const { name, age } = person;`
</KeyConcept>
