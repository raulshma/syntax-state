## How Vite Works: The Magic Explained

Vite uses a clever two-mode approach:

### Development Mode: Native ES Modules

```javascript
// Your browser can now understand imports natively!
// Vite leverages this to skip bundling during development

// index.html
<script type="module" src="/src/main.js"></script>

// main.js - Browser requests this directly
import { createApp } from 'vue'  // Vite transforms on-the-fly
import App from './App.vue'       // Only when requested!
```

<InfoBox type="analogy" title="Like a Smart Waiter">
Think of Vite as a smart waiter. Instead of bringing the entire menu to your table, they only bring what you order. Need the appetizer? Here it is! Want dessert later? They'll get it when you ask. No wasted trips, no waiting for food you haven't ordered yet.
</InfoBox>

## Basic Configuration

Vite works great with zero configuration, but here's what a simple config looks like:

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  
  server: {
    port: 3000,        // Dev server port
    open: true,        // Open browser automatically
  },
  
  build: {
    outDir: 'dist',    // Output directory
  },
})
```

## Vite's Superpowers

### 1. Instant Hot Module Replacement (HMR)

```javascript
// Change this component...
function Button() {
  return <button>Click me!</button>
}

// ...and see the update in ~50ms, not seconds!
```

### 2. Built-in Support for Modern Features

- TypeScript (no config needed!)
- JSX
- CSS Modules
- JSON imports
- Static assets

### 3. Optimized Production Builds

```bash

# Uses Rollup under the hood for optimized bundles
# Automatic code splitting
# CSS extraction
# Asset optimization
```
