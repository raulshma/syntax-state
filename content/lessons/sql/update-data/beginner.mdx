# UPDATE: Changing Existing Data

Learn how to modify records that already exist in your database!

## Ô∏è Basic UPDATE Syntax

```sql
UPDATE table_name
SET column = new_value
WHERE condition;
```

> Ô∏è **Always use WHERE!** Without it, you'll update EVERY row!

##  Your First UPDATE

```sql
-- Change a customer's email
UPDATE customers
SET email = 'newemail@email.com'
WHERE id = 1;
```

##  Updating Multiple Columns

```sql
UPDATE customers
SET
    email = 'alice.new@email.com',
    phone = '555-1234',
    city = 'Los Angeles'
WHERE id = 1;
```

##  Updating with Calculations

```sql
-- Give everyone a 10% raise
UPDATE employees
SET salary = salary * 1.10
WHERE department = 'Engineering';

-- Reduce stock after a sale
UPDATE products
SET stock = stock - 5
WHERE product_id = 100;
```

##  The Importance of WHERE

```sql
-- GOOD: Updates only one customer
UPDATE customers
SET status = 'inactive'
WHERE id = 123;

-- DANGEROUS: Updates ALL customers!
UPDATE customers
SET status = 'inactive';
-- Now every customer is inactive! üò±
```

##  Safe Update Workflow

1. **Write SELECT first** to see what you'll change:

```sql
SELECT * FROM customers WHERE city = 'Boston';
```

2. **Convert to UPDATE**:

```sql
UPDATE customers
SET discount = 0.15
WHERE city = 'Boston';
```

3. **Verify** the changes:

```sql
SELECT * FROM customers WHERE city = 'Boston';
```

##  Common UPDATE Patterns

| Goal                | Example                        |
| ------------------- | ------------------------------ |
| Set specific value  | `SET status = 'active'`        |
| Add to number       | `SET count = count + 1`        |
| Subtract            | `SET stock = stock - quantity` |
| Multiply            | `SET price = price * 1.1`      |
| Clear a value       | `SET notes = NULL`             |
| Set to current date | `SET updated_at = GETDATE()`   |
