# Components and JSX

Welcome to React! React is a powerful library for building user interfaces. Think of it as a way to create interactive, dynamic web pages by breaking them down into smaller, reusable pieces called **components**.

Imagine you're building with LEGO blocks. Each block is a small, self-contained piece that you can combine with other blocks to create something amazing. That's exactly how React components work!

<InfoBox type="tip" title="Why React?">
React makes it easy to build complex user interfaces by breaking them into small, manageable pieces. Each piece (component) handles its own logic and appearance, making your code easier to understand and maintain.
</InfoBox>

## What Are Components?

A **component** is like a LEGO block for your website. Just like LEGO blocks:

- Each component is a **self-contained piece** with its own look and behavior
- Components can be **combined** to build bigger things
- You can **reuse** the same component multiple times
- Components can **contain other components** (nesting)

## Creating Your First Component

In React, a component is just a **JavaScript function** that returns what should appear on screen. Here's the simplest component:

```jsx
function Greeting() {
  return <h1>Hello, World!</h1>;
}
```

That's it! A function that returns some HTML-like code. Let's break it down:

1. **Function name starts with a capital letter** - This tells React it's a component
2. **Returns JSX** - The HTML-like syntax inside the return statement
3. **Can be used like an HTML tag** - `<Greeting />`

<KeyConcept title="Component Names Must Start with a Capital Letter">
React uses the capital letter to distinguish between components (`<Greeting />`) and regular HTML elements (`<div>`). Always capitalize your component names!
</KeyConcept>

## What is JSX?

JSX stands for **JavaScript XML**. It's a special syntax that lets you write HTML-like code inside JavaScript. It looks like HTML, but it's actually JavaScript!

```jsx
// This is JSX
const element = <h1>Hello, World!</h1>;

// It gets transformed into this JavaScript
const element = React.createElement('h1', null, 'Hello, World!');
```

### JSX Rules to Remember

1. **Return a single parent element**
```jsx
// ‚ùå Wrong - multiple elements
return (
  <h1>Title</h1>
  <p>Paragraph</p>
);

// ‚úÖ Correct - wrapped in a parent
return (
  <div>
    <h1>Title</h1>
    <p>Paragraph</p>
  </div>
);
```

2. **Close all tags** (even self-closing ones)
```jsx
// ‚ùå Wrong
<img src="photo.jpg">
<input type="text">

// ‚úÖ Correct
<img src="photo.jpg" />
<input type="text" />
```

3. **Use `className` instead of `class`**
```jsx
// ‚ùå Wrong (class is a reserved word in JavaScript)
<div class="container">

// ‚úÖ Correct
<div className="container">
```

## Using JavaScript in JSX

One of the coolest things about JSX is that you can put JavaScript expressions inside it using **curly braces** `{}`.

```jsx
function UserGreeting() {
  const name = "Alice";
  const age = 25;
  
  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>You are {age} years old.</p>
      <p>Next year you'll be {age + 1}!</p>
    </div>
  );
}
```

### What Can Go Inside `{}`?

- **Variables**: `{name}`
- **Math**: `{2 + 2}`
- **Function calls**: `{getName()}`
- **Ternary expressions**: `{isLoggedIn ? "Welcome" : "Please log in"}`

## Props: Passing Data to Components

**Props** (short for "properties") are how you pass data from a parent component to a child component. Think of them like function arguments!

```jsx
// Parent passes data via props
<Greeting name="Alice" />

// Child receives props
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Or use destructuring (cleaner!)
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}
```

### Props Make Components Reusable

Instead of creating separate components for each user, you create ONE component and pass different data:

```jsx
<UserCard name="Alice" role="Developer" />
<UserCard name="Bob" role="Designer" />
<UserCard name="Charlie" role="Manager" />
```

## Nesting Components

Components can contain other components, just like HTML elements can contain other elements. This is called **composition**.

```jsx
function App() {
  return (
    <div>
      <Header />
      <MainContent />
      <Footer />
    </div>
  );
}

function Header() {
  return (
    <header>
      <Logo />
      <Navigation />
    </header>
  );
}
```

## Key Takeaways

<KeyConcept title="Components = LEGO Blocks">
Components are reusable pieces of UI. Build small components and combine them to create complex interfaces.
</KeyConcept>

<KeyConcept title="JSX = HTML in JavaScript">
JSX lets you write HTML-like code in JavaScript. Use `{}` to embed JavaScript expressions.
</KeyConcept>

<KeyConcept title="Props = Component Arguments">
Props let you pass data to components, making them reusable with different content.
</KeyConcept>
