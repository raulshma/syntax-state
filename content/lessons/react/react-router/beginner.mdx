# React Router (Navigation)

Welcome to React Router! Have you ever wondered how websites show different pages without actually reloading? That's the magic of client-side routing, and React Router makes it easy!

<InfoBox type="tip" title="The Book Chapters Analogy">
Think of your React app like a book. Each **route** is a chapter, and React Router is like the table of contents. When you click a chapter link, you don't get a whole new book - you just flip to that chapter. The URL is like a bookmark that remembers which chapter you're on!
</InfoBox>

## The Problem: Single Page Apps Need Navigation

In a traditional website, clicking a link loads an entirely new page from the server. But React apps are "Single Page Applications" (SPAs) - there's only one HTML page!

So how do we show different content for different URLs?

```jsx
// Without React Router - Manual and messy! üò´
function App() {
  const [page, setPage] = useState('home');
  
  return (
    <div>
      <nav>
        <button onClick={() => setPage('home')}>Home</button>
        <button onClick={() => setPage('about')}>About</button>
      </nav>
      
      {page === 'home' && <Home />}
      {page === 'about' && <About />}
      {/* URL doesn't change, can't bookmark, back button doesn't work! */}
    </div>
  );
}
```

<KeyConcept title="What is Client-Side Routing?">
Client-side routing lets you change what's displayed based on the URL, without requesting a new page from the server. The URL updates, the browser history works, and only the necessary components re-render!
</KeyConcept>

## The Solution: React Router

React Router handles all the URL magic for you:

```jsx
// With React Router - Clean and powerful! üéâ
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>
      
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
```

## The Core Components

### 1. BrowserRouter - The Container

Wrap your entire app with `BrowserRouter`. It enables routing throughout your application:

```jsx
import { BrowserRouter } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      {/* Your app goes here */}
    </BrowserRouter>
  );
}
```

### 2. Routes and Route - The Map

`Routes` is like a switch statement for URLs. It looks at the current URL and renders the matching `Route`:

```jsx
import { Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </BrowserRouter>
  );
}
```

### 3. Link - The Navigation

Use `Link` instead of `<a>` tags for navigation. It prevents full page reloads:

```jsx
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      {/* ‚úÖ Use Link for internal navigation */}
      <Link to="/">Home</Link>
      <Link to="/about">About</Link>
      
      {/* ‚ùå Don't use <a> for internal links */}
      {/* <a href="/about">About</a> - This causes a full reload! */}
    </nav>
  );
}
```

## Dynamic Routes: URLs with Variables

Sometimes you need URLs like `/products/123` or `/users/alice`. The number or name changes, but the page structure is the same.

React Router uses `:paramName` to capture these dynamic parts:

```jsx
// Route definition
<Route path="/products/:productId" element={<ProductDetail />} />

// This matches:
// /products/1
// /products/42
// /products/awesome-widget
```

Inside your component, use `useParams` to get the value:

```jsx
import { useParams } from 'react-router-dom';

function ProductDetail() {
  const { productId } = useParams();
  
  return <h1>Product #{productId}</h1>;
}
```
