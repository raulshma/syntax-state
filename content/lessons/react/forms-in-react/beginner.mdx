# Forms in React

Welcome to forms! Forms are how users interact with your application - signing up, logging in, searching, and more. In React, we handle forms a bit differently than regular HTML.

<InfoBox type="tip" title="The React Way">
In React, we use "controlled components" where form inputs are controlled by React state. This gives us full control over the form data!
</InfoBox>

<ProgressCheckpoint section="introduction" xpReward={15} />

## What Are Controlled Components?

In regular HTML, form inputs manage their own state internally. In React, we take control by:

1. Storing the input value in React state
2. Updating state when the input changes
3. Setting the input's value from state

This creates a "single source of truth" - React state is always the authority on what the input contains.

```jsx
// The controlled component pattern
function NameInput() {
  const [name, setName] = useState('');  // 1. State holds the value
  
  return (
    <input
      value={name}                        // 3. Input shows state
      onChange={(e) => setName(e.target.value)}  // 2. Changes update state
    />
  );
}
```

## Your First Controlled Input

Let's build a simple controlled input step by step:

```jsx
import { useState } from 'react';

function GreetingForm() {
  // Step 1: Create state for the input value
  const [name, setName] = useState('');
  
  // Step 2: Create a handler for changes
  const handleChange = (event) => {
    setName(event.target.value);
  };
  
  return (
    <div>
      {/* Step 3: Connect input to state */}
      <input
        type="text"
        value={name}
        onChange={handleChange}
        placeholder="Enter your name"
      />
      
      {/* The greeting updates automatically! */}
      <p>Hello, {name || 'stranger'}!</p>
    </div>
  );
}
```

## Handling Different Input Types

React handles different input types similarly, with a few small differences:

### Text Inputs

```jsx
<input
  type="text"
  value={text}
  onChange={(e) => setText(e.target.value)}
/>
```

### Number Inputs

```jsx
<input
  type="number"
  value={age}
  onChange={(e) => setAge(Number(e.target.value))}
/>
```

### Checkboxes (use `checked` instead of `value`)

```jsx
<input
  type="checkbox"
  checked={isSubscribed}
  onChange={(e) => setIsSubscribed(e.target.checked)}
/>
```

### Select Dropdowns

```jsx
<select value={color} onChange={(e) => setColor(e.target.value)}>
  <option value="red">Red</option>
  <option value="blue">Blue</option>
  <option value="green">Green</option>
</select>
```

## Handling Form Submission

When a form is submitted, we need to:
1. Prevent the default browser behavior (page reload)
2. Process the form data

```jsx
function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  
  const handleSubmit = (event) => {
    event.preventDefault();  // Stop page reload!
    
    // Now do something with the data
    console.log('Logging in with:', email, password);
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />
      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
      />
      <button type="submit">Log In</button>
    </form>
  );
}
```

<KeyConcept title="Always Prevent Default!">
Without `event.preventDefault()`, the browser will reload the page when the form submits - losing all your React state!
</KeyConcept>

## Key Takeaways

<KeyConcept title="Controlled Components">
In React, form inputs should be "controlled" - their value comes from state, and changes update state.
</KeyConcept>

<KeyConcept title="The onChange Handler">
Every controlled input needs an onChange handler that updates state with the new value.
</KeyConcept>

<KeyConcept title="Prevent Default on Submit">
Always call `event.preventDefault()` in your form's onSubmit handler to prevent page reloads.
</KeyConcept>
