# Variable Naming Rules: Syntactic and Semantic Guidelines

Understanding JavaScript's naming rules goes beyond just avoiding syntax errors. This lesson covers the complete set of rules and industry-standard conventions.

## JavaScript Identifier Rules

A **variable identifier** must follow specific syntax rules defined by the ECMAScript specification.

### Valid Identifier Characters

| Position        | Allowed Characters                                   |
| :-------------- | :--------------------------------------------------- |
| First character | Letters (a-z, A-Z), underscore (\_), dollar sign ($) |
| Subsequent      | All above + digits (0-9)                             |

```javascript
// Valid identifiers
let name; // Letter start
let _private; // Underscore start
let $element; // Dollar sign start
let camelCase123; // Mix of valid characters

// Invalid identifiers
// let 123abc;      // Cannot start with digit
// let my-var;      // Hyphen not allowed
// let my var;      // Spaces not allowed
// let class;       // Reserved word
```

## Reserved Words

JavaScript reserves certain words for language features. These cannot be used as identifiers.

### Always Reserved:

```javascript
// Keywords (always reserved)
const keywords = [
  "break",
  "case",
  "catch",
  "continue",
  "debugger",
  "default",
  "delete",
  "do",
  "else",
  "finally",
  "for",
  "function",
  "if",
  "in",
  "instanceof",
  "new",
  "return",
  "switch",
  "this",
  "throw",
  "try",
  "typeof",
  "var",
  "void",
  "while",
  "with",
];

// ES6+ additions
const es6Keywords = [
  "class",
  "const",
  "let",
  "super",
  "yield",
  "export",
  "import",
  "extends",
  "static",
];

// Literals
const literals = ["null", "true", "false"];
```

### Future Reserved (Strict Mode):

```javascript
"use strict";

// These are reserved in strict mode:
// implements, interface, package, private, protected, public
// let implements = 1; // SyntaxError in strict mode
```

### Contextually Reserved:

```javascript
// These are reserved only in certain contexts:
let await; // OK in non-async context
// let await; // Error inside async function

async function test() {
  // let await = 1; // SyntaxError!
  // let yield = 1; // SyntaxError in generators!
}
```

## Naming Conventions by Context

### Variables and Functions: camelCase

```javascript
// Variables
let userName = "alice";
let itemCount = 5;
let isAuthenticated = false;

// Functions
function getUserById(id) {}
function calculateTotalPrice(items) {}
function validateEmailFormat(email) {}
```

### Constants: SCREAMING_SNAKE_CASE

```javascript
// Configuration constants
const MAX_RETRY_ATTEMPTS = 3;
const API_BASE_URL = "https://api.example.com";
const DEFAULT_TIMEOUT_MS = 5000;

// Note: This is for truly constant values
// Regular const variables use camelCase
const userPreferences = { theme: "dark" }; // Object can change
```

### Classes and Constructors: PascalCase

```javascript
class UserAccount {}
class ShoppingCart {}
class DatabaseConnection {}

function LegacyConstructor() {} // Old-style constructors
```

### Private Conventions:

```javascript
// Underscore prefix for "private" (convention only)
class Example {
  _privateField; // Convention: don't access externally
  #reallyPrivate; // ES2022: True private field

  _privateMethod() {} // Convention
  #truePrivate() {} // True private
}
```

## Semantic Naming Patterns

### Boolean Variables

```javascript
// Prefix with is, has, can, should, will
let isActive = true;
let hasPermission = false;
let canEdit = true;
let shouldRefresh = false;
let willExpire = true;

// Avoid
let active = true; // Less clear it's a boolean
let permission = false; // Ambiguous
```

### Array Variables

```javascript
// Use plural nouns
let users = [];
let orderItems = [];
let selectedIds = [];

// Describe the collection
let activeUsers = users.filter((u) => u.isActive);
let completedOrders = orders.filter((o) => o.status === "complete");
```

### Functions and Methods

```javascript
// Action verbs for functions
function getUserData() {} // get - retrieves
function setUserStatus() {} // set - assigns
function updateProfile() {} // update - modifies
function deleteRecord() {} // delete - removes
function validateInput() {} // validate - checks
function parseJSON() {} // parse - transforms
function renderComponent() {} // render - displays

// Event handlers
function handleClick() {}
function onSubmit() {}
function beforeUpdate() {}
```
