## What is CI/CD? 

Let's break down the acronyms:

**CI = Continuous Integration**
- Developers merge code frequently (daily or more)
- Each merge triggers automatic builds and tests
- Catches problems early, when they're easy to fix

**CD = Continuous Delivery / Deployment**
- **Delivery**: Code is always in a deployable state
- **Deployment**: Code is automatically deployed to production

## Simple Automation Example

```yaml
# A simple CI/CD pipeline
name: Build and Deploy

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4       # Get code
      - run: dotnet restore             # Get packages
      - run: dotnet build               # Compile
      - run: dotnet test                # Verify
      - run: dotnet publish -o ./app    # Package
  
  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - run: echo "Deploying to server..."  # Ship it!
```
