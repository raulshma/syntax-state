## Advanced Meta Tags

Beyond title and description, there are many meta tags that help search engines and browsers understand your content.

### Viewport Meta Tag

Essential for mobile-friendly pages:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

This tells browsers how to scale your page on different devices. Without it, mobile users see a tiny desktop version.

### Robots Meta Tag

Control how search engines interact with your page:

```html
<!-- Allow indexing (default) -->
<meta name="robots" content="index, follow">

<!-- Prevent indexing -->
<meta name="robots" content="noindex, nofollow">

<!-- Index but don't follow links -->
<meta name="robots" content="index, nofollow">
```

<InfoBox type="warning" title="Use noindex Carefully">
Adding `noindex` removes your page from search results entirely. Use it only for pages you don't want found, like thank-you pages or admin areas.
</InfoBox>

## Open Graph Protocol

Open Graph (OG) tags control how your page appears when shared on Facebook, LinkedIn, and other platforms.

```html
<meta property="og:title" content="Learn Web Development">
<meta property="og:description" content="Master coding with interactive lessons">
<meta property="og:image" content="https://example.com/og-image.png">
<meta property="og:url" content="https://example.com/learn">
<meta property="og:type" content="website">
<meta property="og:site_name" content="MyLearningPrep">
```

### Essential Open Graph Tags

| Tag | Purpose | Best Practice |
|-----|---------|---------------|
| `og:title` | Share title | 60-90 characters |
| `og:description` | Share description | 200 characters max |
| `og:image` | Preview image | 1200x630 pixels |
| `og:url` | Canonical URL | Full absolute URL |
| `og:type` | Content type | website, article, product |

<KeyConcept title="Images Make Shares Stand Out">
Posts with images get significantly more engagement. Always include an og:image that's visually appealing and represents your content.
</KeyConcept>

## Twitter Card Tags

Twitter has its own meta tags for controlling how links appear in tweets:

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@yourusername">
<meta name="twitter:title" content="Learn Web Development">
<meta name="twitter:description" content="Master coding with interactive lessons">
<meta name="twitter:image" content="https://example.com/twitter-image.png">
```

### Card Types

| Type | Description | Image Size |
|------|-------------|------------|
| `summary` | Small square image | 144x144 min |
| `summary_large_image` | Large rectangular image | 300x157 min |

<InfoBox type="tip" title="Fallback Behavior">
If Twitter tags aren't set, Twitter will use your Open Graph tags. You only need Twitter-specific tags if you want different content for Twitter.
</InfoBox>

## Canonical URLs

When the same content exists at multiple URLs, use the canonical tag to tell search engines which version is the "official" one:

```html
<link rel="canonical" href="https://example.com/original-page">
```

### When to Use Canonical Tags

- **URL parameters**: `?ref=twitter` vs `?ref=facebook`
- **HTTP vs HTTPS**: Always point to HTTPS
- **WWW vs non-WWW**: Pick one and stick with it
- **Duplicate content**: Product pages accessible via multiple categories

```html
<!-- On https://example.com/products?category=shoes&color=red -->
<link rel="canonical" href="https://example.com/products/red-shoes">
```

<KeyConcept title="Canonicals Prevent Duplicate Content Penalties">
Without canonical tags, search engines might see your content as duplicated across multiple URLs, which can hurt your rankings.
</KeyConcept>

## Complete Head Section Example

Here's a well-optimized `<head>` section:

```html
<head>
  <!-- Basic Meta -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Learn Web Development | MyLearningPrep</title>
  <meta name="description" content="Master web development with interactive lessons and AI-powered learning paths.">
  
  <!-- Canonical -->
  <link rel="canonical" href="https://learn.raulshma.xyz/learn">
  
  <!-- Open Graph -->
  <meta property="og:type" content="website">
  <meta property="og:url" content="https://learn.raulshma.xyz/learn">
  <meta property="og:title" content="Learn Web Development | MyLearningPrep">
  <meta property="og:description" content="Master web development with interactive lessons.">
  <meta property="og:image" content="https://learn.raulshma.xyz/og-image.png">
  <meta property="og:site_name" content="MyLearningPrep">
  
  <!-- Twitter -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:url" content="https://learn.raulshma.xyz/learn">
  <meta name="twitter:title" content="Learn Web Development | MyLearningPrep">
  <meta name="twitter:description" content="Master web development with interactive lessons.">
  <meta name="twitter:image" content="https://learn.raulshma.xyz/og-image.png">
  
  <!-- Favicon -->
  <link rel="icon" href="/favicon.ico">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">
</head>
```

## Testing Your Meta Tags

### Tools to Validate

1. **Facebook Sharing Debugger** - Debug Open Graph tags
2. **Twitter Card Validator** - Test Twitter cards
3. **Google Rich Results Test** - Check structured data
4. **LinkedIn Post Inspector** - Preview LinkedIn shares

<InfoBox type="info" title="Cache Clearing">
Social platforms cache your meta tags. After making changes, use their debugging tools to force a refresh of the cached data.
</InfoBox>
